# App icon + splash — finishing steps (run these on your Mac)

> **Status: DONE (2026-06-10).** The Liquid Glass icon shipped without needing the
> Icon Composer GUI — the manifest schema was validated empirically against Xcode
> 26.4's `actool` (key discovery: **`groups` is ordered top-first**, and the parent
> circles need `blend-mode: screen` for the glowing overlap). The finished bundle
> lives at `TwoOfUs/TwoOfUs.icon` (schema notes in
> `design/icon/TwoOfUs.icon/README.md`), `project.yml` points
> `ASSETCATALOG_COMPILER_APPICON_NAME` at it, and it's verified on the simulator
> (light + dark home screen, splash hand-off). The steps below are kept for
> reference — e.g. if you later want to hand-tune materials in Icon Composer.

This change shipped everything that can be done headlessly: the new **"Two of Us"**
mark (two overlapping parent circles with a warm baby light at the heart), wired into
the PNG `AppIcon.appiconset` (so the app already builds and shows the new icon), the
`LaunchLogo` for the static launch screen, an animated SwiftUI `SplashView`, and a
scaffolded Liquid Glass `.icon` bundle.

What's left needs a Mac, because **Icon Composer is a macOS GUI app** and `xcodegen` /
`xcodebuild` / the Simulator aren't available in the web environment. Steps below.

> The app targets **iOS 26.0 only**, so the known "`.icon` back-deploys with
> inconsistent rendering on older iOS" caveat does **not** apply to us.

---

## 1. Finish the Liquid Glass icon in Icon Composer

Artwork is pre-generated at `design/icon/TwoOfUs.icon/Assets/`:

| Layer (bottom → top) | File | What it is |
|---|---|---|
| Background | `background.png` | near-black radial-depth plate |
| Parents | `parent-left.png` (periwinkle), `parent-right.png` (teal) | the two of us |
| Baby | `baby.png` | warm `#FFF4E8` point of light (intentionally **not** amber) |

1. Open **Icon Composer** (ships with Xcode 26): `xcrun open -a "Icon Composer"`,
   or Xcode ▸ Open Developer Tool ▸ Icon Composer.
2. Try opening `design/icon/TwoOfUs.icon` directly. The `icon.json` there is a
   **best-effort starter** (Apple doesn't publish the manifest schema), so if it
   doesn't open cleanly: create a **New** icon and **drag the four PNGs from
   `Assets/`** into the layer list, bottom-to-top in the order above. The artwork is
   the durable part; the manifest is disposable.
3. Tune the glass material per group: enable **specular** on the parent circles and
   the baby, add a soft **shadow**, and dial **translucency** so the parents read as
   glass and the overlap glows. Keep the baby a bright, clean highlight.
4. Cycle the appearance previews — **Default / Dark / Clear (light+dark) / Tinted** —
   and confirm the mark stays legible at small sizes in each. The tinted render keys
   off luminance, which the artwork already accounts for.
5. **Save** the bundle as `TwoOfUs.icon`.

---

## 2. Wire the `.icon` into the build

The Xcode project is generated by **XcodeGen** from `project.yml` (the `.xcodeproj`
is gitignored, regenerated by `ci_scripts/ci_post_clone.sh`). So make the change in
`project.yml`, not in Xcode's UI.

1. Put the saved `TwoOfUs.icon` inside the app's sources folder, e.g.
   `TwoOfUs/TwoOfUs.icon` (the target already pulls in everything under `TwoOfUs/`).
2. In `project.yml`, point the app-icon build setting at it:

   ```diff
       settings:
         base:
           PRODUCT_BUNDLE_IDENTIFIER: com.taylorseale.twoofus
   -       ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon
   +       ASSETCATALOG_COMPILER_APPICON_NAME: TwoOfUs
   ```

3. Regenerate and open:

   ```sh
   xcodegen generate
   open TwoOfUs.xcodeproj
   ```

4. Build to a simulator and confirm the icon renders with Liquid Glass. **Once
   confirmed,** you can delete the now-redundant fallback
   `TwoOfUs/Assets.xcassets/AppIcon.appiconset` (or leave it — it's harmless and
   keeps a flat PNG around for tooling/marketing).

---

## 3. Verify on the Simulator / device

- **Icon:** home screen, Settings ▸ Two of Us, Spotlight, App Library. Toggle the
  system appearance and the home-screen **tint** to check the dark/tinted/clear looks.
- **Splash hand-off:** cold-launch the app. The static launch screen (the centered
  mark on black) should hand off **seamlessly** into `SplashView` — the mark is drawn
  at the same 240pt centered size, so there should be no jump. You'll see a gentle
  glow + single heartbeat on the baby, the "Two of Us" wordmark fades up, then it
  cross-fades into the app (~1.5s total).
- **Reduce Motion:** Settings ▸ Accessibility ▸ Motion ▸ Reduce Motion **on**. The
  splash should skip the pulse and just fade in/out quickly (~0.85s).
- **Demo-mode regression:** open the app, enter and exit Demo mode. The splash must
  **not** replay on the container swap (it's hosted above the `.id(containerToken)`
  rebuild in `TwoOfUsApp.swift`).

## Regenerating the artwork

All artwork comes from one source. To tweak proportions or colors, edit the constants
near the top of `design/icon/generate_icons.py` (`CIRC_R`, `CIRC_SHIFT`, `DOT_RADIUS`,
`BABY`, palette) and run:

```sh
python3 design/icon/generate_icons.py
```

That rewrites the `.icon` layer PNGs, the `AppIcon.appiconset` fallback, and the
`LaunchLogo` together. If you change the mark's geometry, mirror it in
`CradleMark` in `TwoOfUs/Features/Shared/SplashView.swift` (the SwiftUI mark uses the
same ratios) so the splash keeps matching the icon.
